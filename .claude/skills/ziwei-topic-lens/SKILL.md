# 紫微斗数 Topic Lens Skill

> v3.1 (2026-05-23) — 以星定极: 论物时可由 用户指定物象锚点星，以该星所在宫为太极
> v3.0 (2026-05-22) — 太极点选择器 + 宫对信号索引
> v3.0 核心变更: 索引层与推演层分离。lens 负责 factual 信号盘点（宫对索引 + 命主交互 + 变动汇总），core 只做 interpretive 推演。
> v2.0 变更: 新增 topic_tag 输出字段
> 不限流派，只做太极点框架 + factual 信号索引

## 职责

**做**：
- 判断用户问的是谁 / 什么事 / 什么物
- 选主太极点 + 理由
- 选次太极点 + 用途 + 权重
- **宫对索引 (Palace Pair Index)**: 以轴(宫对)为最小单位，列出涉及宫位的完整信号 [v3.0 新增]
- **命主↔太极点交互**: 命飞化入太极 + 太极飞化入命 [v3.0 新增]
- **重要飞化落点**: 忌/禄落宫的完整信号 [v3.0 新增]
- **变动信号汇总**: 驿马位/自化/视同自化/组合索引命中 [v3.0 新增]
- 列禁用误区（Banned Misreads）
- 排飞化链优先级（Primary / Secondary / Background）
- 生成 source lookup 查询关键词

**不做**：
- 不输出断语 / 不做象核 / 不做 Phase C 取象
- 不写 reader prose
- 不做忌义分读（那是 core 的 Phase D）
- 组合索引只列命中/不命中，不解读含义

## 输入

- 用户问题（必需）
- **chart-stage1.md**（必需 — v3.0 提升为必需，宫对索引需要读盘面数据）
- **sihua-transaction-map.md**（必需 — v3.0 新增，命主交互和飞化落点需要交易图）
- **chart-facts.yaml**（必需 — v3.0 新增，变动信号需要 chart-facts）
- cross-topic-structural.md（可选，有则引用理则审计辅助 Background）

## 工作流

### Step 1: 解析问题

确定论人/论事/论物：
- 论人：问的是哪个人群？标记"人位=是"，后续强制多角度
- 论事：问的是什么事？
- 论物：问的是什么物？

### Step 2: 选太极点

查 `references/` 目录有无现成参照。有则加载并微调；无则从头选。

**Step 2a: 以宫定极（论人/论事，默认路径）**

主太极点选择原则：
- 问谁就用谁的宫：路人=交友，熟人=兄弟，伴侣=夫妻...
- 问什么事就用什么宫：事业=官禄，钱=财帛，健康=疾厄...
- 不混用

**Step 2b: 以星定极（论物，用户指定锚点星）**

当用户问的是一个具体的**物**（一台车、一笔钱、一份合同、一个空间……），太极点不一定对应固定宫位，而是对应盘上某颗星：

1. **用户指定物象锚点星** — 哪颗星代表问的这个物，由 用户凭直觉定。CC 不自动映射。
   - 如 用户说"待定"或不确定 → 暂停，问 用户确认再继续。
   - CC 可以提示"这个物可能对应X星或Y星"作为参考，但**不替用户 做决定**。
2. **找星所在宫** — 锚点星在 chart-stage1 里坐哪个宫，该宫即为主太极点。
3. **后续走正常 pipeline** — 宫对索引、命主交互、飞化落点、变动汇总照常走。

⚠️ **以星定极的边界**：
- 锚点星只决定太极点在哪个宫。进入该宫之后，判读靠飞化，不靠星的静态标签。
- 禁止"X星+Y星=某物"的 cookbook 直断。星组合只定物的质感，吉凶成败由四化决定。
- 同一颗星可能代表不同的物（天机=车/机器/道路/思路），具体是哪个由问题语境 + 用户判断。

次太极点：与主太极点有直接飞化关系的宫位。标用途+权重+限制。

### Step 3: 确定涉及宫位 + 构建宫对

列出本 topic 涉及的所有宫位，组成宫对(轴):

**必列轴**:
- 太极点轴: 太极宫 ↔ 对宫（任何 topic 都列）
- 命主轴: 命宫 ↔ 迁移宫（人位 topic 必列；事位 topic 看命↔太极有无飞化交互决定）

**条件轴**:
- 忌落点轴: 太极宫干忌落宫 ↔ 其对宫（如落宫不在上述轴内）
- 禄落点轴: 太极宫干禄落宫 ↔ 其对宫（如落宫不在上述轴内）

轴的确定原则: **如果两个宫之间有视同自化/对宫化入/空宫看对宫/飞化碰撞等结构关联，它们必须成对出现**。

### Step 4: 宫对信号盘点 (Palace Pair Index)

对每个轴的两宫，逐项读 chart-stage1.md 填写:
- 地支位（**必填**，不可空）
- 主星+旺度
- 辅星+旺度（关注左辅右弼"再一次"、天马驿马、煞星、空劫）
- 组合名（查 source-router combo dispatch 表）
- 生年象
- 自化（宫干化X在本宫）
- **视同自化**（宫干化X在对宫）— **必须从发射宫视角标**
- 对宫化入（对宫干化X入本宫）

然后填轴内飞化（从交易图读）+ 轴内组合索引（查 RC-xu2-ch4-08 等）。

### Step 5: 命主↔太极点交互

从交易图读:
- 命宫干飞化中，有哪些落入太极点/太极对宫/太极关联宫？
- 太极点宫干飞化中，有哪些落入命宫/迁移宫？
- 碰撞检查: 飞化落点有无生年/自化象？标碰撞类型。
- 双向关系定性: 互飞忌/互飞禄/双向科/单向等。

### Step 6: 重要飞化落点

太极点宫干忌/禄的落宫，如不在已列轴内 → 补列该宫完整信号。

### Step 7: 变动信号汇总

扫描所有涉及宫位，汇总变动信号:
- 驿马位(寅申巳亥)
- 自化/视同自化
- 组合索引命中（如天同+昌曲+忌="云消雾散"）
- 辅星变动（左辅"再一次"、天马、空劫）

标总数。

### Step 8: 链优先级 + 禁用误区 + 人位多角度

- 排链优先级（Primary / Secondary / Background）
- 列禁用误区
- 如人位: 列多角度提示（关系面/利益面/画像面的初步方向，不做解读）

### Step 9: Source lookup queries + Topic tag

## References

`references/` 目录下每个文件对应一个常见主题的太极点参照。

## 核心纪律

1. **不做断语** — lens 是框架+索引，不是结论
2. **宫对是最小索引单位** — 不列散装单宫，列宫对(轴) [v3.0]
3. **视同自化必须从发射宫视角标** — 宫干化X在对宫=视同自化 [v3.0]
4. **地支位必填** — 驿马位/桃花位/库位是变动/桃花/收藏的基础信号 [v3.0]
5. **组合索引必查** — RC-xu2-ch4-08 文昌文曲 + RC-xu2-ch4-07 辅曜煞曜 [v3.0]
6. **人位宫必标多角度** — 关系面+实际利益面+人物画像面 [v3.0]
7. **太极点选择有理由** — 每个选择都标明依据
8. **禁用误区必须列** — 防止下游 skill 犯已知错误
9. **链优先级分三层** — Primary > Secondary > Background
10. **不替代 source-lookup** — 只生成查询关键词，不直接引用 RC cards

## First-invocation Flow

1. 接收用户问题
2. 确认 chart-stage1 + sihua-transaction-map + chart-facts 存在（v3.0 必需）
3. 检查 references/ 有无匹配参照
4. 走 Step 1-9
5. 输出 Topic Lens Packet v3
6. 不继续，不产 finding
