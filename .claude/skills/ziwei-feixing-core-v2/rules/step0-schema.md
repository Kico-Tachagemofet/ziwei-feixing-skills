# Step 0: 飞化交易象全列 Schema

> 从 SKILL.md v3.3 外置。Step 0 产出 sihua-transaction-map.md 时遵循此 schema。
> 位置: Stage 1.5 完成后、Stage 2 之前
> 性质: 纯 factual layer，等同于占星 aspect table。不做解读，只做结构记录。
> 前置: chart-stage1 + audit PASS + cross-topic-structural.md + chart-facts.yaml

## 产出内容

**A. 48 edges (十二宫 × 四化)**

对每一宫的宫干，逐条列出四化:

```
| 化 | 星 | 落宫 | 入/照/冲 | 落宫已有象? | 碰撞? |
```

入=飞化落入该宫；**照=禄权科到对宫辉映(正面)**；冲=**忌**到对宫冲击(负面)。⚠️ 每条 edge 必须同时记录"入X宫"和"照/冲X对宫"，不可只记入不记照。禄权科入A宫=照A对宫；忌入A宫=冲A对宫。落宫已有象=该宫已有生年/自化？碰撞=飞化碰既有象→标六组类型+是否同星。

**B. Collision Index (碰撞索引)**

哪些宫接收了多条飞化？哪些宫同时被禄和忌命中？

```
| 承受宫 | 接收的飞化 (来源宫→化象) | 碰撞类型 |
```

碰撞类型: 禄忌交会 / 多忌汇聚 / 禄权科聚 / 忌冲+禄入 / ...

**C. Hub Stats (枢纽统计)**

- 最多出度的宫 (发射最多有效交易的)
- 最多入度的宫 (承受最多飞化的)
- 忌的汇聚点 (哪些宫被 ≥2 条忌命中/冲?)
- 禄的汇聚点 (哪些宫被 ≥2 条禄命中?)

**D. 因果組 (禄随忌走)**

识别因果組模式:
- A宫忌入B宫，A宫禄入C宫 → B的困境与C的出路来自同一个A
- 追忌找禄：先找忌落点，再追同发射宫的禄落点
- 追禄找忌：先找禄落点（得），再问代价（同宫忌去哪）

```
| 因果組 | 忌端(A忌→B) | 禄端(A禄→C) | 含义方向 |
```

**E. 格局 Candidates (结构级模式)**

从交易图直接可见的格局级信号:
- 忌冲命 (理则三)
- 禄忌同宫/对宫 — 得失绑定
- 自化与飞化方向一致/相反
- 来因宫结构特征
- 枢纽宫 (接收 ≥3 条飞化)

**F. 生年四化 overlay**

生年四化在交易图上的位置标注:
- 哪些飞化链的落点碰到了生年四化？
- 哪些飞化链经过了生年四化所在宫？
- 生年忌所在宫的出度/入度 (它发射什么? 它被什么命中?)

## 硬约束

1. **纯 factual** — 不写"所以命主..."，不做解读，不选主次
2. **全列** — 不跳宫，不省略"看起来不重要"的化
3. **含义方向只写一层** — 因果組的"含义方向"只写"B困→C出路来自A"，不写"所以命主应该..."
4. **不需要 deep cards** — 这是结构层，用短卡 + chart data 即可
5. **生年四化用不同标记** — 生年四化是体，飞化是用，两者在图上用不同标记
