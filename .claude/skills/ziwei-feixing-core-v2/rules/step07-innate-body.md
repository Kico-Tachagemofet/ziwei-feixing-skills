# Step 0.7: 先天体 (Innate Body)

> 位置: Step 0 (交易图) 之后、所有 Stage 2 topic 之前
> 性质: 半结构化描述层。读 deep card 但不出断语。
> 输出: `innate-body.md`
> 前置: chart-stage1 + sihua-transaction-map.md

## 职责

先天体是**所有 topic 的共享输入**。它回答"这个人先天带了什么"，不回答"这个人怎么样"。

**做**:
- 命迁线 combo + deep card 必读 → 内外画面
- 生年四化 4 落宫: combo + 旺度 + 一句话含义
- 来因宫结构特征
- 从交易图引用结构速写
- 标 open threads

**不做**:
- 不出断语 / 不做象核 / 不选太极点
- 不读小星 (后续 topic + wtz addon 的事)
- 不做忌义分读 (那是 Stage 2 Phase D)
- 不写 reader prose

## 硬约束

1. **combo deep 全读** — 命迁线 + 生年四化 4 落宫的 combo 全读 deep。先天体是共享输入，底层画面不能省。
2. **宫象合参** — 四化含义必须过宫位落地。参照 SKILL.md 四化回查清单。
3. **不做断语** — "禄在疾厄=体质好"是描述。"命主体质不错"是断语。先天体只做前者。
4. **不选太极点** — 那是 topic-lens 的职责。
5. **忌汇聚必查结构必然性** — 引用 feixing-minimal-grammar 忌汇聚规则，标明是否高频忌星坐该宫。
6. **先天体是共享输入** — 产出后给所有后续 topic 引用。topic 层面不重做命迁线和生年四化分析。
