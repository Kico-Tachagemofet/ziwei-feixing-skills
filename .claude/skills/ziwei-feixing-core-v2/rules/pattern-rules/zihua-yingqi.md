# P14 自化应期

## trigger
- 盘面存在自化或视同自化。
- 用户问题涉及何时、几岁、哪年、哪月、应期、时间点。

## rc-sources
- RC-xu3-ch1-11
- RC-xu3-ch1-13
- RC-xu3-ch6-11
- RC-xu3-ch4-17

## 操作步骤
1. 标出自化宫 A 与自化象 X；若是视同自化，记录箭头指向本宫还是对宫。
2. 找同类生年 X 所在宫 B；自化权找生年权，自化忌找生年忌，自化禄找生年禄，自化科找生年科。
3. 列候选时间：A 成为流命或相关流宫时；B 成为流命或相关流宫时；A 与 B 在大运/流年/流月交会时。
4. 若同盘有媒介象，媒介象宫可定更细时间；若三合宫参与，另一三合宫可作时间提示。
5. 流年有自化时，该年有变化；论月、日、时常偏后段，或取本宫/对宫后段。
6. 输出主应点与备选点，不用单一绝对年份覆盖所有同盘可能。

## 判读规则
- 自化一定有时间观念，但流年多用于应期，不负责重新决定有无。
- 自化宫与同类生年宫交会时气最强，可作为最终应期。
- 大运先定十年空间，流年定年份，流月/流日/流时再细化。

## 输出格式
```yaml
pattern: P14
name: 自化应期
self_palace: A宫
self_symbol: 权
same_birth_palace: B宫
primary_timing: "A/B 交会的大运或流年"
backup_timing: ["A成流命", "B成流夫", "媒介象宫成流月"]
notes: "流年自化则该年有变化，月日偏后段"
```

## 边界
- 没有自化时，不用本卡硬定应期。
- 自化应期不是现实保证；只输出盘内气象强弱和候选窗口。
- 健康、寿元、婚姻重大断语必须另有主题规则支持。
