# Provenance Policy

## 体系位置

- `[wtz]` = 王亭之《安星法及推断实例》小星/安星/实例信号。
- `[xu]` = `ziwei-feixing-core-v2` 的许铨仁飞星主链。

`[wtz]` 只补星曜质感，不替换 `[xu]` 的四化、飞化、自化、忌义、应期主框架。

## 允许

- 用 `[wtz]` 判断辅曜、佐曜、煞曜、空劫、桃花、文曜、科名诸曜的信号域。
- 用 `[wtz]` 校验安星法、流曜、起例、星曜分类。
- 用 `[wtz]` 给已成立的飞化链补充形相、人物、物象、事件质感。
- 用 `[wtz]` 标记成对星、单星、会照、三方四正、流限叠会的辅助意义。

## 禁止

- 不可用小星静态分布绕过飞化链直接产 finding。
- 不可用中州推断框架覆盖 core 的许铨仁飞星理则。
- 不可把“星曜等级”当成重要性排序；王亭之强调低级别小星也可能改写星系性质。
- 不可把桃花、煞曜、空曜标签直接翻成吉凶，必须回到宫位、飞化链、流限与组合。
- 不可把小星取象写成独立等号，例如“红鸾=幽默”“文曲=绕弯”。必须写出它修饰哪颗主星、哪组同宫/对宫组合、哪条飞化链。

## 冲突处理

1. 四化/飞化结构与 `[wtz]` 小星信号一致：小星可升为 supporting signal。
2. 四化/飞化结构没有连接：小星降级为 background。
3. 小星信号与飞化结构冲突：标 `conflict_note`，以飞化结构为主。
4. 只有小星无结构：不产断语，只列待核。

## 校准

若用户提供断事级校准，标 `[kico-calibration]`，并记录成“组合语境规则”，不要写成单星本义。
