# Inference Principles Deep

> card_type: deep
> source: [wtz]

## 原则

1. 不按星曜等级轻重来决定是否有用。低等级杂曜也可能改变星系性质。
2. 成对星要先检查是否成对，再判断助力是否完整。
3. 单星在父母、夫妻等六亲宫尤其敏感，可能提示两重、缺配或关系不完整。
4. 小星只说明质感：贵助、文书、移动、刑伤、桃花、空耗、孤寡。
5. 事件成立仍要回到宫位、飞化链、流限、主星组合。
6. 断事级取象如果依赖同宫/对宫主星，必须存为组合语境，不可回填到单星本义。

## Role Ladder

| role | 条件 |
|---|---|
| primary | 小星组合被题目直接询问，且有宫位/流限/飞化承接 |
| supporting | 与飞化链方向一致，增强质感 |
| background | 只有静态分布，没有推导关系 |
| excluded | 与 topic、宫位、流限都无关 |

## 输出约束

- 每条小星信号都要写 `path`：星曜如何连接 topic。
- 每条小星信号都要写 `base_combo`：小星修饰的是哪颗主星或哪组组合。
- 每条小星信号都要写 `boundary`：不能由此推出什么。
- 若借用实例模式，只能说“类似触发结构”，不要把历史案例当作必然模板。
