# 紫微斗数 Render Checklist

> 内容层规则。写每段前过一遍。

## 1. 生活判断测试

每句实质内容必须能被真人回答"准"或"不准"。

✓ "你心思特别容易放在伴侣/关系方向——先天就有一股力把你拽向那边"
✗ "夫妻宫的忌象形成了一种收束性的能量定位"

测试方法：把这句话念给一个不懂紫微的朋友听，他能不能说"对，我就是这样"或"不对，我不是"？

## 2. Chart-tech 分层

**正文**零术语。以下内容只能出现在每小节末尾 `技术依据:` 行（inline，不折叠）：
- 化禄/化权/化科/化忌
- 飞宫、理则、四化、八大理则
- 生年象、自化、视同自化
- RC-ID、Phase A/B/C/D
- 干支代号（丙、戊、辛、丙辰、戊午等）
- Finding 编号（F-PART-001 等）
- 飞化链完整写法（命(壬)忌武曲→交友）、edge#、CG-ID、combo名

**正文**允许出现（已进入日常）：
- 宫名（命宫、夫妻宫——但不带干支）
- 大运/流年（说"26到35岁这十年"不说"虚岁26~35"）
- "先天"/"后天"（作为时间概念）

**正文**允许出现（星性描写层 — v1.3 新增）：
- 星名 + 原书白话定性（"破军在福德=此生难得清闲""廉贞天府=老谋深算、府城深"）
- 小星名 + 具体角色（"天月是病星""天哭=身体不好时情绪化""天空=空想"）
- 组合通性人话版（"动中求财积蓄成富""桃花+耳软心活+孤僻专断"）
- 飞化链人话版（"你命宫把困扰的能量送去了朋友圈""感情方向往事业送了一笔延续力和一笔困扰力"）

这些是盘面辨识度的骨架——去掉它们渲染就变成性格测试。

**正文**禁止出现（v1.3 新增）：
- **旺度判断**（"旺地抗忌""庙地正面发挥""陷=力量弱"等）— 许铨仁飞星派不主看旺度，旺度在体系内有争议，不作正文判据。技术依据行可作事实记录。

~~旧规范: `<details>` 折叠技术块~~ — 已弃用，改为 inline `技术依据:` 行。

## 3. 来源宫上下文

渲染时必须保持 finding 的来源宫上下文。

✓ "感情里的纠葛不往外爆，往内收——每次感情有波折，内心都要经历一轮拆毁重建"
✗ "家宅方面有一些不安定"（丢了"因为感情"这个来源）

原则：读者读完应该知道"为什么"这个领域有这个现象，而不只是"有"。

## 4. Confidence 语气对应

| confidence | 散文语气 | 示例 |
|---|---|---|
| high | 直陈 | "你天生有……" |
| medium-high | 偏直陈带一丝余地 | "大概率……"、"通常表现为……" |
| medium | 对冲 | "倾向于……"、"很可能……" |
| low | 明确保留 | "这一点不太确定" |
| gap | 不写或诚实 | "这方面信息不够" |

## 5. R1 校准体现

R1 验前事说准的 → 可以直陈
R1 说不准的 → 不写进正文（已校准排除）
R1 说部分准的 → 只写准的那部分

示例：R1说"想太多对，计划赶不上变化不对" → 散文只写"想太多"，不写"计划赶不上变化"。

## 6. 不改 finding 方向

| finding 说 | 渲染可以 | 渲染不能 |
|---|---|---|
| "无红灯" | 说"这十年感情没有必坏的底层格局" | 暗示"其实有隐患" |
| "有凶底色但有缓冲" | 两面都写，缓冲不省略 | 只写凶不写缓冲 |
| "忌入不是忌冲" | 说"不是往外翻脸的那种" | 按忌冲的严重度去写 |
| confidence: low | 用对冲语气 | 直陈得像 high 一样 |

## 7. Source Gap 诚实

有 source gap 的 finding，渲染时：
- 可以写，但语气降级
- 或者在段尾加一句"这一点的依据不太充分"
- 不能伪装成高信心判断

## 8. 跨 topic 重复控制

同一飞化链在多个 topic 都产了 finding（如庚忌=天同→迁移冲命 同时出现在 career 和 spiritual），渲染时：
- 各 topic 从自己角度写，措辞不同
- 不要复制粘贴同一段话
- 可以在一个 topic 交叉引用另一个（"这一点在精神层面也有对应"）

## 9. 段落节奏 (v1.3 修订)

- **每个 finding 独立小节，每小节 3-6 段**。不压缩不合并——5 个 finding 写 5 个小节
- 每段 3-6 句话
- 开头总结 1-2 句抓核心
- ~~旧: 每个 topic 2-4 段，不超过 5 段~~ — Z04-anonymous 翻车: 4 finding 压成 3-4 段变性格测试，用户 反馈"看着像性格测试不像解盘"
- 不写"综上所述"类收尾——最后一段本身就是收束

## 10. 时间表述

| 原始数据 | 渲染表述 |
|---|---|
| 虚岁 29 | "今年"或"28岁左右" |
| 大运 26~35 | "26到35岁这十年" |
| 流年 2026 | "今年" |
| 大运 16~25 | "十几二十岁那十年" |
| 生年四化 | "先天"/"天生" |

## 11. 高对比度结构识别 + 结构化展示 (v1.4 新增)

当盘面有 **high-contrast 信号** (paradox / 0 互飞 / 体之体 / 来因宫=对宫 / RC 反查 N/N 全缺/全中 / 多信号同向共振 等), reader 散文不要糊过去, 用**结构化展示**让 reader 直接看到结构强度:

### 11a. "N 线共振"清单 (≥3 个独立信号指向同一结论时)

✓ **格式**:
```
配偶"不善沟通"在本盘有 5 条独立信号汇聚:
1. **紫微无吉辅** = 孤君条件 → 不善表达深层
2. **七杀** = 性刚不爱说软话
3. **天哭** = 哀感/情感受困/有苦不易说
4. **龙池陷** = 才艺(含表达才艺)不显
5. **对宫天府+陀罗+天马** = 信息流被阻碍

→ 5 线共振 = "想说说不出来"
```

✗ 错误: "配偶不善沟通, 紫微无吉辅+七杀+天哭+龙池陷+对宫陀罗都说明这一点" (堆在一句里看不清)

### 11b. RC 反查对照表 (体系明确的"N 条信号"结构成立/缺失时)

✓ **格式** (HTML 或 markdown 表):
```
许铨仁体系里有一套"成婚保留信号"——4 条信号是否成立:
| 信号 | 本盘 |
| 夫视同自化禄=有缘 | 缺失 |
| 夫忌入父母=有名分 | 缺失 |
| 夫权入命=媒介事件 | 缺失 |
| 夫科入田宅=最后质不会离婚 | 缺失(关键)

→ 4/4 全缺 = 离婚倾向结构性成立
```

✗ 错误: "盘面缺这缺那, 所以离婚易" (没有 RC 对照感, reader 不知道你判断的依据)

### 11c. Cross-topic 同结构 stripe 引用

✓ **格式**: "这一条在'婚姻与配偶'那一节也对应——配偶因素冲到父母关系, 这里又看到career 关键宫(父母)的压力"

✗ 错误: 7 个 topic 各说各的, 同一结构在不同 topic 重复描述但没有显式 cross-ref

### 触发条件 (什么时候用 #11 写法)

- composition.md §2 paradox 主锚 → 必须有 N 线共振或结构化展示
- composition.md §5 张力点 → 用对照表展示张力两面
- RC-XX-NN 引用的"N 条信号成立"型结构 → 用反查对照表
- 5+ 颗小星同宫的场景合成 → 用 "X 系" 分组共振写法 (3 组光华/空虚/孤损 等)
- 多 finding 指向同一结论 → 用编号清单展示信号汇聚

### 风险 (滥用 #11 的副作用)

- ❌ 不要每段都用清单, 散文节奏会被破坏
- ❌ "N 线共振" 用于 < 3 个信号 → 像凑数
- ❌ 对照表用于没有明确 RC 基准的判断 → 假装权威
- ✓ 每个 topic 1-2 处结构化展示足矣, 其余仍用散文
